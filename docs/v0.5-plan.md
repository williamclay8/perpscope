# v0.5 Plan: Compatibility Report Export

v0.5 should make PerpScope more useful to terminal builders who are not ready to wire the adapter yet.

## Goal

Paste or drop decoded Percolator output, then export a portable compatibility report that can be attached to GitHub issues, shared with terminal teams, or saved next to fixtures.

## User Story

As a terminal builder, I want to paste one decoded output shape and download a compact compatibility JSON so I can see which fields PerpScope maps, which fields are missing, and which fields were ignored without copying screenshots or raw protocol logs.

## Scope

- Add `exportCompatibilityReport(input, snapshot?)` or equivalent helper in the adapter package.
- Include `shape`, `status`, `score`, `recognizedSections`, `missingFields`, `ignoredFields`, source provenance, package version, and generated timestamp.
- Add a cockpit download action inside capture intake.
- Redact or reject secret-like and mutating fields using the existing read-only safety checks.
- Add a fixture example under `examples/`.
- Add tests for export shape stability and safety rejection.
- Document the export in `README.md` and `docs/terminal-builder-quickstart.md`.

## Out Of Scope

- wallet connection
- signing
- transaction simulation
- order payload validation
- trade recommendations
- private user account exports

## Acceptance Criteria

- `npm run check` passes.
- Exported report is valid JSON.
- Exported report includes package version and source provenance.
- Export rejects wallet, signer, transaction, instruction, order, private key, seed, mnemonic, and API key fields.
- Live cockpit can generate a report from `examples/percolator-cli.bundle.json`.
- Docs show how to attach the report to the decoded-shape issue form.

## Follow-Up Ideas

- Compare two compatibility reports and show adapter drift.
- Convert ignored fields into candidate alias suggestions.
- Add a small CLI wrapper for report generation.
